Database 基礎

我帶你認識 Database 嘅基本概念、分類同埋管理系統

Database 係一個非常廣闊嘅概念

電子化儲存嘅數據

我想先同你講清楚，Database 呢個詞其實係一個非常廣闊嘅概念。簡單嚟講，只要係電子化儲存嘅數據，都可以叫做 Database。

好多人一聽到 Database 就會諗起好複雜嘅系統，但其實你每日用緊嘅好多嘢都係 Database。我哋睇下幾個常見例子：

Postgres

傳統關係型數據庫，用表格形式儲存數據，有嚴格嘅結構要求

DynamoDB

Amazon 嘅 NoSQL 數據庫，無固定結構，擴展性好強

Google Sheets

你冇睇錯，試算表都係一種 Database，只係比較簡單

甚至筆記簿

從廣義角度睇，有組織咁記錄數據嘅工具都係 Database

我想同你強調

Database 唔一定要好複雜、好高科技。只要係電子化儲存數據，無論係 Excel、Google Sheets 定係專業嘅 Postgres、MongoDB，本質上都係 Database。重點係佢哋點樣幫你組織同管理數據。

唔係所有 Database 都保證 ACID

SQL vs NoSQL 嘅核心分別

好多人以為所有 Database 都會保證某啲特性，例如原子性（Atomicity）、一致性（Consistency）。但我要同你講，呢個係一個常見嘅誤解。

ACID 合規只係其中一個類別。大部分 SQL 數據庫都係 ACID 合規，但 NoSQL 數據庫就唔一定會強制執行 ACID。

  • 1 SQL 數據庫（例如 Postgres、MySQL）：大部分都係 ACID 合規，保證數據嘅完整性同一致性。適合需要強一致性嘅場景，例如銀行系統、交易平台。
  • 2 NoSQL 數據庫（例如 DynamoDB、MongoDB）：唔強制執行 ACID，換取更高嘅擴展性同效能。適合需要處理大量數據、高並發嘅場景。
  • 3 最終一致性：NoSQL 好多時只保證最終一致性，即係話數據可能會有短暫嘅唔一致，但最終會達到一致狀態。

我嘅建議

唔好假設所有 Database 都有相同嘅保證。SQL 同 NoSQL 係兩個唔同嘅世界，各有優勢。你要根據實際需求去選擇：需要強一致性就用 SQL，需要高擴展性就考慮 NoSQL。

Database 同 DBMS 嘅分別

數據 vs 管理軟件

我想同你解釋清楚一個技術上嘅分別。嚴格嚟講，Database 只係指數據本身，而 DBMS（Database Management System）先係管理呢啲數據嘅軟件。

不過喺日常交流入面，大部分人都會直接叫 DBMS 做 Database。例如講 Postgres，其實係指緊 DBMS，但大家都會簡單咁叫佢做 Database。

  • 1 Database 係數據本身：所有儲存落去嘅資料，例如用戶資料、訂單記錄、產品庫存等等，呢啲先係真正嘅 Database。
  • 2 DBMS 係管理軟件：負責處理查詢（Query）、插入（Insert）、刪除（Delete）等操作，仲要確保數據嘅安全性同完整性。
  • 3 DBMS 提供介面：俾開發者同應用程式可以透過 SQL 或者其他查詢語言嚟存取同操作數據。
  • 4 常見 DBMS 例子：Postgres、MySQL、MongoDB、DynamoDB 呢啲都係 DBMS，但我哋日常都簡稱佢哋做 Database。

查詢功能

DBMS 俾你可以用 SQL 或其他語言嚟搵出你需要嘅數據

數據操作

處理插入、更新、刪除等操作，確保數據正確咁儲存

安全控制

管理用戶權限、數據備份、災難恢復等安全措施

效能優化

索引管理、查詢優化，確保系統高效運行

我嘅結論

雖然技術上 Database 同 DBMS 係兩樣嘢，但喺實際工作入面，你唔使太執著呢個分別。當你講 Postgres 或者 MySQL 嘅時候，大家都明你係講緊成個系統。重點係要明白 DBMS 嘅角色：佢唔只係儲存數據，更加係提供一整套工具嚟管理、查詢同保護你嘅數據。
